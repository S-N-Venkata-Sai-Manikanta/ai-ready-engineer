For Trainers
Delivery Guide — How to Teach This Module

Recommended teaching flow, classroom techniques and assessment approach for trainers.

01 · Before Class
Pre-Class Preparation
  • Verify every learner has Git installed and a GitHub account before Session 1
  • Prepare a "broken merge" repository ready for live conflict-resolution drills
  • Pre-create an organisation so learners can practice team PR workflows from Day 2
  • Have GitHub Copilot trial activated on the demo VS Code instance
02 · In Class
Live Delivery Style
  • Teach Git on the terminal first — only show GUI tools after fundamentals stick
  • Run every command live on screen, never hide it behind GUI buttons
  • Every learner makes at least one commit during every session
  • Use the classroom GitHub organisation for real PRs between learners
03 · Engagement
Student Engagement Tactics
  • Run a "Merge Conflict Battle" — learners deliberately create and resolve conflicts in pairs
  • Assign code-review rotations so every learner reviews someone else's PR weekly
  • Track "first contributor" badges for the first 5 open-source PRs
  • Showcase the best GitHub profile READMEs in class each week
04 · Assessment
Assessment Approach
  • Evaluate the learner's public GitHub profile — pinned projects, README quality, commit history
  • Grade PR descriptions and review comments, not just code
  • Run a timed "fix a real open-source bug" challenge for the final assessment
  • Verify learners have made at least one real external open-source contribution
