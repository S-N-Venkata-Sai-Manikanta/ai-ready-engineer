For Trainers
Delivery Guide — How to Teach This Module

Recommended teaching flow, classroom techniques and assessment approach for trainers.

01 · Before Class
Pre-Class Preparation
  • Set up shared OpenAI, Anthropic and Gemini accounts with budget caps to control cost
  • Provide every learner with a starter Python repo, .env template and dependencies installed
  • Pre-record a "first AI app in 5 minutes" demo as a fallback if APIs are slow on the day
  • Have a tested function-calling weather demo ready for Session 6
02 · In Class
Live Delivery Style
  • Make the first API call live in the terminal — print raw JSON before showing libraries
  • Always show the API cost and token count on screen so learners build cost intuition
  • Live-stream responses so learners feel the "ChatGPT-like" magic in their own code
  • Use real product examples to teach system prompts — show how startups structure them
03 · Engagement
Student Engagement Tactics
  • Run a "Lowest Token Cost" challenge for the same task — sharpens prompt economy
  • Have learners demo their chatbot live to peers, who try to break it
  • Compare OpenAI vs Claude vs Gemini outputs on the same prompt every session
  • Encourage learners to ship one tiny AI feature into a real app each week
04 · Assessment
Assessment Approach
  • Evaluate the support-bot project on prompt design, conversation memory and error handling
  • Grade the multimodal tool on integration quality — three APIs working in one flow
  • Run a 2-hour timed assignment: build an AI feature with function calling from a spec
  • Review production-readiness — retries, logging, cost guards, fallback model logic
