For Trainers
Delivery Guide — How to Teach This Module

Recommended teaching flow, classroom techniques and assessment approach for trainers.

01 · Before Class
Pre-Class Preparation
  • Set up Cursor AI, Bolt, Lovable, Replit and v0.dev with logged-in accounts on demo machine
  • Connect Vercel and GitHub accounts in advance — deploys must be instant on stage
  • Prepare an "idea bank" of 10 small app ideas for the live build moments
  • Have a starter Supabase project pre-configured with demo data ready to plug in
02 · In Class
Live Delivery Style
  • Open Day 1 with a 10-minute "zero to deployed app" demo to spark excitement
  • Always type the prompt live — let learners suggest words as you go
  • When the tool generates wrong code, walk through the debug conversation on screen
  • End each session with a public deployment URL learners can browse on their phone
03 · Engagement
Student Engagement Tactics
  • Run a "Build War" — 45 minutes, two learners build competing versions of the same app
  • Reward the first learner to ship a working deployed link each session
  • Pair-program — one types prompts, the other reviews AI output, then swap
  • Maintain a class showcase wall with every learner's deployed URL
04 · Assessment
Assessment Approach
  • Grade on a live deployed URL, not on local screenshots
  • Evaluate prompt history and iteration quality, not just the final output
  • Score MVP project on functionality, polish and originality of feature set
  • Run a 60-minute timed "build a new feature" challenge for the final assessment
